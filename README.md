# Clippy

A native macOS clipboard manager that keeps a large, instantly-searchable
history while idling in the **tens of MB of RAM** — not the ~1 GB that managers
like Maccy can reach. Low memory and rich history are not a tradeoff here; they
fall out of the architecture.

Measured: **3,300 clips (300 images) → 13 MB idle `phys_footprint`.**
Reproduce it yourself: `Scripts/mem_benchmark.sh`.

## How the memory promise is kept

- **Metadata-only in-memory model.** The history list holds lightweight
  `ClipMetadata` values — a short preview, scalar fields, and on-disk paths.
  Never image bytes, never full text bodies. A list of thousands is a few
  hundred KB regardless of clip size.
- **Content lives on disk.** Text in SQLite, image originals as external files.
  Full content is loaded on demand (paste / full preview) and released right
  after. History depth is disk-limited, not RAM-limited.
- **Thumbnails are downsampled at capture** via ImageIO
  (`CGImageSourceCreateThumbnailAtIndex`) — a 20 MP screenshot never gets fully
  decoded into memory.
- **Bounded LRU thumbnail cache**, flushed when the panel closes, so memory
  returns to baseline after capture/paste spikes.
- **Virtualized list** (`LazyVStack`) so scrolling renders only visible rows.

## Build & run

Requires the Swift toolchain (Command Line Tools are enough — no full Xcode).

```bash
swift build -c release          # compile
Scripts/build_app.sh release    # → build/Clippy.app  (ad-hoc signed)
Scripts/make_dmg.sh             # → build/Clippy.dmg  (drag-to-Applications)
Scripts/mem_benchmark.sh        # repeatable MEM-01 footprint benchmark
```

## Install (DMG)

1. Open `Clippy.dmg` and drag **Clippy** to **Applications**.
2. Launch it. Clippy is a background menu-bar agent — look for the clipboard
   icon in the menu bar, not the Dock.

### First launch on Apple Silicon (Gatekeeper)

Clippy is ad-hoc signed (no paid Apple Developer ID yet). On **macOS Sequoia and
later**, Control-click → Open no longer bypasses Gatekeeper. Instead:

1. Double-click Clippy. macOS will say it can't verify the developer.
2. Open **System Settings ▸ Privacy & Security**, scroll to **Security**, and
   click **Open Anyway** next to the Clippy message.
3. Confirm once more. Clippy launches and is trusted from then on.

> Note: because the build is unsigned/ad-hoc, the Accessibility grant (below) is
> tied to this exact binary and is reset if you replace the app with a fresh
> unsigned rebuild. Developer-ID signing + notarization (v2 / DIST-03) removes
> this friction.

### Accessibility permission (for auto-paste)

To paste back into the app you were using, macOS requires the **Accessibility**
permission. Clippy prompts on first run. If you skip it, selecting a clip still
copies it to the clipboard — you just press ⌘V yourself. Enable it any time at
**System Settings ▸ Privacy & Security ▸ Accessibility**.

## Using Clippy

| Action | Shortcut |
| --- | --- |
| Open history panel | **⌘⇧V** |
| Search | just type |
| Move selection | ↑ / ↓ |
| Paste selected | **return** |
| Paste as plain text | **⌥return** |
| Add / remove from paste stack | **⌘return** |
| Paste whole stack in sequence | **return** (while stack non-empty) |
| Paste recent item N | **⌘1 … ⌘9** |
| Cycle type filter (All/Text/Images/Files) | **⇥** |
| Paste with transform (trim, case, JSON, …) | **⌘T** |
| Edit tags / collection | **⌘G** |
| Save selected clip as a snippet | **⌘S** |
| Toggle full-content preview | **⌘Y** |
| Pin / unpin | **⌘P** |
| Delete item | **⌘⌫** |
| Dismiss panel | **esc** |

Search supports tokens: `app:safari` filters by source app, `tag:work` by
collection. Click the **Snippets** chip (or ⌘S to save one) for reusable text
with `{date}` / `{time}` / `{clipboard}` placeholders.

All shortcuts (open, pin, delete, preview) are rebindable in **Settings**, and
clear / pause-capturing / *Launch at Login* live in the menu-bar menu.

## Settings (Maccy-parity)

Open with **⌘,** from the menu bar. Five panes:

- **General** — launch at login, rebindable shortcuts, search mode
  (**exact / fuzzy / regex / mixed**).
- **Appearance** — panel position (center / cursor / menu-bar / last), thumbnail
  height, app icons, match highlighting, footer & search-field visibility,
  recent-copy-in-menu-bar.
- **Storage** — per-type save toggles (text / images / files), sort
  (last / first copied / number of copies), max items & age retention.
- **Ignore** — ignore by application, by pasteboard type, or by regular
  expression (beyond the built-in password-manager blocklist).
- **Advanced** — pause capturing, full-content preview toggle, clipboard
  check interval.

History is **never** cleared on quit — only explicit delete / clear removes
items.

## Privacy

- Content marked `org.nspasteboard.{Concealed,Transient,AutoGenerated}Type` is
  never read or stored.
- A built-in blocklist (1Password, Bitwarden, LastPass, KeePassXC, Keychain
  Access, …) means clipboard contents from those apps are never recorded.

## Where data lives

`~/Library/Application Support/Clippy/`
- `clippy.db` — SQLite history + FTS5 full-text index
- `images/` — full-resolution image originals
- `thumbnails/` — downsampled previews

Retention prunes unpinned clips by count (default 10,000) and age (default 90
days). Pinned items are always exempt.
